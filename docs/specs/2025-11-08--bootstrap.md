# Bootstrap: инвентаризация репозитория и базовые операционные документы

Дата: 2025-11-08
Автор: Warp (Senior Full‑Stack Engineer & Terminal Operator)

Цели
- Провести инвентаризацию локальных git‑репозиториев в рабочем каталоге и сформировать отчёт.
- Обновить/создать архитектурный обзор проекта.
- Подготовить runbook с гайдлайнами по GitHub/Git‑потокам и Railway/Cloudflare.
- Зафиксировать изменения в CHANGELOG.

Не‑цели
- Внесение изменений в код/схему БД.
- Конфигурация реальных секретов или деплой.

Область и входные допущения
- inventory_root: C:\dev (может быть скорректирован владельцем репозитория при необходимости).
- Искать репозитории по наличию каталога .git.
- Метрики инвентаризации: ветка по умолчанию (origin/HEAD), remotes, последние 3 коммита, наличие ключевых конфигов (Next.js, тесты, деплой‑артефакты).

Артефакты (выход)
- docs/ops/repo-inventory.md — сводная таблица и выводы.
- docs/arch/overview.md — актуальный архитектурный обзор.
- docs/ops/runbook.md — гайды по GitHub/Git, Railway/Cloudflare.
- docs/CHANGELOG.md — запись о выполненных работах.

Риски
- Потенциально большой объём обхода диска — ограничиваемся C:\dev.
- Отсутствие некоторых утилит (gh, wrangler, railway) — даём инструкции, не исполняем.

План валидации
- Просмотреть сгенерированные документы, проверить ссылки/пути.
- Для репозитория текущего проекта проверить соответствие инвентаризации фактическому состоянию.

План отката
- Удалить созданные файлы из docs/* при необходимости (только документация, без изменений кода/конфигов).
